# FFID Banner QA Tools — Product Context

## Users
QA specialists on the FFID LiveOps team. They scan banner lists daily, compare sheet data against CDN status and in-game state. Sessions are long, repetitive, and detail-heavy.

## Register
**Product.** Internal operations tool. Design serves the task: fast scanning, accurate status reading, minimal distraction.

## Brand voice
Calm, clinical, utilitarian. No marketing language. Labels state facts.

## Anti-references
- Purple/blue gradients on dark backgrounds
- Glassmorphism and backdrop-blur panels
- Oversized rounded cards (rounded-xl everywhere)
- Neon accent colors (orange `#ff6b35` hero buttons)
- Decorative emoji-style status icons without labels
- Centered marketing-style hero layouts

## UX priorities
1. **Scannability** — tabular data, aligned columns, monospace CDN paths
2. **Hierarchy** — summary metrics → filters → grouped sections → rows
3. **Status clarity** — semantic color + text label, never color alone
4. **Density** — show more rows per screen without clutter
5. **Focus** — sticky context bar; filters always visible while scrolling
